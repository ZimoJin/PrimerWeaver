Golden Gate Assembly Primer Designer

The Golden Gate Assembly module enables the design of primers for seamless modular DNA assembly using Type IIS restriction enzymes (typically BsaI) and T4 DNA ligase. This tool automates the creation of unique 4 bp overhangs that allow multiple DNA fragments to be assembled in a defined order and orientation in a single one-pot reaction without intermediate purification steps. Golden Gate is particularly well-suited for standardized biological parts libraries, synthetic biology applications, high-throughput genetic circuit construction, and iterative design workflows.

Quick Start Guide

  1. Upload or paste your destination vector and insert sequences (maximum of 6 fragment) in FASTA or plain text format into the corresponding input fields, in the desired assembly order. Select the Type IIS enzyme, verify the automatically detected vector backbone, and adjust design parameters if necessary.
  2. Click "Design Primers" to generate primers with integrated overhang sequences. Review the primer table, overhang compatibility matrix, and predicted assembly schematic. Download the assembled FASTA file to verify that the insert orientation matches the intended design. If the orientation is incorrect, reverse-complement the insert sequence and regenerate the primers

Golden Gate Assembly Principle

Golden Gate assembly employs Type IIS restriction enzymes that recognize a specific DNA sequence but cleave outside the recognition site. This unique property allows the generation of predictable, customizable overhangs that enable directional, seamless ligation of multiple DNA fragments in a single reaction. Crucially, the restriction enzyme recognition sites are removed during assembly, leaving no scars in the final construct.

Overhang Design Principle

The core strength of Golden Gate lies in the design of unique, non-interfering overhangs. Each fragment pair junction requires a unique 4 bp overhang that is not present elsewhere in the assembly:

  • 4 bp Overhangs: Type IIS enzymes with 1 bp offset (e.g., BsaI) naturally generate 4 bp sticky ends, which provide sufficient specificity while maintaining assembly simplicity
  • Directional Assembly: The forward overhang on one fragment must be complementary to the reverse overhang on the adjacent fragment; non-complementary overhangs cannot ligate, preventing misassembly
  • Seamless Junctions: After assembly, the Type IIS recognition sites flanking the overhangs are removed, resulting in scar-free junctions between fragments

Input Format

Both vector and insert inputs accept FASTA, plain DNA text, or standard GenBank files (.gb, .gbk, .gbff, .genbank) with LOCUS and ORIGIN blocks. The short name in LOCUS is used as the sequence title. Only one sequence is allowed per input field; if multiple FASTA headers are detected, only the first sequence will be used. If you upload plain sequence without a > header, the file name (without extension) is used as the name. Proprietary binary formats (e.g. SnapGene .dna) must be exported as GenBank or FASTA first.

PrimerWeaver currently supports four built-in Type IIs restriction enzymes. Once a vector sequence is provided, the system automatically identifies vector features and displays the positions of all Type IIs restriction enzyme sites. The vector map can be rotated to facilitate visualization of the targeted cloning region. The selected vector backbone fragment also determines the automatic 4-bp boundary overhangs used for the first insert left end and the last insert right end.

Fragment Management

  • Add/Remove Fragments: Add multiple DNA sequences for multi-fragment assembly (2–10 fragments recommended; larger assemblies become more complex)
  • Reorder Fragments: Adjust assembly order using interface controls; order determines final construct architecture
  • Fragment Names: Auto-extracted from FASTA headers for clear primer identification; used in output reports
  • Reverse Complement: Toggle individual fragments if needed to match desired orientation and assembly direction
  • Internal overlap entry: For every internal junction, paired Left/Right overhang (custom, 4 bp) fields are provided. These two fields mirror the same junction value. Leave them blank to let the algorithm choose sequence-derived overhangs automatically, or enter a custom exactly 4-bp value to override the automatic choice for that junction.

Design Parameters

Please select the preferred Type IIs restriction enzymes from the dropdown list or search by enzyme name. Once restriction enzymes are selected, the system automatically identifies the longest digestion fragment as the default destination vector backbone. If the longest fragment does not correspond to the intended vector backbone, you may manually select another fragment. That backbone choice directly controls the automatic boundary overhangs assigned to the first and last insert ends; these boundary values are always taken from the selected backbone interpretation rather than from manual typing. Please review the primer design parameters before proceeding, and modify them if necessary.

Results and Output

After primer design, the Golden Gate Assembly module generates a comprehensive set of results to facilitate experimental validation and downstream cloning.

Primer Table

Designed primer pairs are reported in a structured table, including primer sequences (5′→3′), length, GC content, and melting temperatures (Tm). Tm values are displayed for both the annealing core region and the full-length primer, allowing users to assess amplification compatibility. Primer quality metrics, including homopolymer runs, hairpin structures, self-dimer, and cross-dimer formation, are evaluated and flagged to highlight potential risks that may affect PCR performance. Type IIS recognition sites and overhang sequences are highlighted within the primer sequences for clarity.

Assembled diagram and Overhang table

An assembled plasmid schematic is generated to illustrate the cloning workflow, including vector linearization, insert incorporation, and final plasmid assembly. This diagram provides a conceptual overview of fragment orientation and junction structure. A overlap table displays the overhang pairs at each junction, confirming that adjacent fragments have complementary sticky ends. This table provides a quick visual verification that the assembly architecture is correct and that no overhang conflicts or mismatches are present.

Simulated Agarose Gel Electrophoresis

PrimerWeaver generates a simulated agarose gel image to predict experimental outcomes for key cloning steps. The gel includes:

  • A selectable DNA ladder (default: NEB 1 kb Plus DNA Ladder)
  • Undigested vector
  • Type IIs restriction-digested vector
  • PCR-amplified fragments (one lane per fragment, showing expected size)
  • Assembled plasmid

Predicted band sizes are annotated alongside the gel, enabling users to compare expected fragment sizes with experimental results and quickly identify potential cloning issues.

Primer Design Algorithm

Primer Structure

Each Golden Gate primer consists of three functional regions:

  • Annealing core (3′): template-complementary region that drives PCR annealing (typically 18–30 bp)
  • Overhang Region (5′): Contains the Type IIS restriction recognition site and a customizable 4 bp overhang sequence (typically 11~12 bp)
  • Protective base (5′ flanks): short bases upstream of the restriction site to improve digestion efficiency (commonly 2–6 nt; enzyme-dependent)

Design Process

Golden Gate primer design in PrimerWeaver is divided into three linked steps: vector-end interpretation, junction overhang assignment, and primer-tail construction for each insert. These steps are summarized below.

Vector-End Interpretation

The vector is first normalized and scanned for the selected Type IIS restriction enzyme. Unlike standard restriction cloning, Golden Gate requires a vector context that can expose defined 4-bp assembly overhangs after Type IIS digestion. The module computes candidate digest fragments, auto-selects the longest one by default, and allows the user to override that backbone choice. The selected backbone fragment is then used to derive the left and right vector overhangs from the cut geometry of the chosen Type IIS enzyme. Those vector-derived values become the fixed boundary conditions for the first insert left overhang and the last insert right overhang, and the interface overwrites those two boundary fields automatically whenever vector/enzyme/backbone interpretation changes.

If the vector does not provide usable end overhangs under this interpretation, the design does not immediately fail. Instead, the module records that vector-derived ends are unavailable and falls back to a seamless junction rule at the assembly boundaries.

Junction Overhang Assignment

After vector constraints are defined, the algorithm assigns a 4-bp overhang to each insert-insert or vector-insert junction. For each internal junction, the module first checks whether the user has entered a custom junction value in the paired Right overhang / Left overhang fields. If a non-empty custom value is present, it must be exactly 4 bp and is used directly for that junction. If both paired fields are blank, the algorithm falls back to sequence-derived candidates, typically using the last 4 bp of the upstream insert and the first 4 bp of the downstream insert. These native candidates are filtered to reject invalid Golden Gate overhangs such as homopolymer 4-mers or palindromic 4-mers.

The module then evaluates complete combinations of junction overhangs at the assembly level, not one junction at a time. Overhang sets are accepted only if the complete panel remains unique even after considering reverse-complement partners. When a fidelity matrix is available for the selected enzyme, PrimerWeaver ranks valid combinations by predicted ligation fidelity and uses the highest-scoring compatible set. If fidelity scoring is unavailable, the module falls back to the first valid non-conflicting combination. A user-supplied internal 4-bp junction therefore acts as a fixed constraint on the global overhang-combination problem rather than as a cosmetic annotation.

Primer-Tail Construction and Assembly Reconstruction

Once the left and right overhangs for each insert are fixed, the module designs the annealing core at the insert termini and then constructs the full primers by appending the Type IIS recognition site, spacer bases, and the selected 4-bp overhangs as 5′ non-annealing tails. During this step, PrimerWeaver removes redundant overlap sequence at both insert ends. At the left end, it trims any largest suffix-prefix duplication between the chosen 4-bp overhang and the start of the forward annealing core. At the right end, it trims any largest duplication between the end of the insert sequence covered by the reverse-primer core and the beginning of the selected right-side overhang. For example, if a custom left overhang ends with ATG and the insert core already begins with ATG, or if the insert already ends with AATG and the chosen right overhang starts with AATG, the shared sequence is not duplicated in the final primer tail. The full Golden Gate primer therefore follows this structure:

5′ protective clamp + Type IIS site + offset bases + selected 4-bp overhang + annealing core 3′

After primer construction, the module reconstructs the assembled plasmid by combining the digested vector backbone with the insert block according to the derived vector cut positions and the chosen junction overhang logic. When vector-derived ends are available, the backbone is re-extracted from the Type IIS cut positions and used as the physical backbone for assembled-sequence reconstruction. If not, the module falls back to a simpler seamless concatenation model for reporting assembled sequence length and downstream visualization.

Edge-Case Logic in the Algorithm

Algorithm edge case Current behavior in Golden Gate module
Vector has zero or one Type IIS site The module does not attempt a normal Golden Gate backbone interpretation. Instead, it reports N/A-style assembly output because the vector cannot provide the two-sided Type IIS digest context needed for standard Golden Gate assembly.
Vector-derived end overhangs cannot be recovered cleanly If explicit vector end overhangs cannot be derived, the algorithm falls back to a seamless boundary rule at the first and last junctions instead of abandoning the whole design immediately.
User supplies an internal 4-bp junction manually A non-empty internal custom overhang is treated as a fixed junction choice. The paired left/right UI fields mirror the same junction value, and the global overhang search must accommodate that preselected 4-bp constraint.
Internal custom junction is partially filled If an internal overlap field is non-empty but not exactly 4 bp, the module blocks the design with a warning dialog rather than silently guessing or padding the junction.
No single junction overhang can be chosen independently The algorithm evaluates overhang combinations across the full assembly panel, because a locally acceptable overhang may still conflict with another junction or its reverse complement elsewhere in the construct.
Fidelity matrix is unavailable The module still completes design by falling back to the first valid non-conflicting overhang combination, but it can no longer rank combinations by predicted ligation fidelity.
Selected overhang is already present at the insert terminus During primer construction, the module checks for the largest shared overlap between the selected 4-bp overhang and the insert sequence already covered by the primer core. This is evaluated at both the left boundary and the right boundary, so shared sequence is not duplicated even when only part of the 4-bp overhang is already present at the insert start or insert end.
Internal Type IIS sites exist in inserts The algorithm can still construct primers, but the design is flagged because internal enzyme sites may fragment inserts during digestion and therefore undermine the intended Golden Gate assembly topology.
Fragment order changes the full overhang solution Because junction overhangs are assigned from neighboring fragment sequence context, reordering inserts changes not only one junction but the global overhang-combination problem and can produce a different final primer set.

Warnings and Edge Cases

The Golden Gate Assembly module uses the shared MW (Message Warning) framework for preflight validation and confirmation dialogs whenever the common visualization layer is available. Some warnings appear before design begins, whereas others are interpreted only after the module inspects vector topology, Type IIS sites, and candidate overhang combinations.

General warning

These warnings mostly come from shared input validation and from high-level Golden Gate preflight checks. In most cases, the user can click Cancel to revise the setup or OK to continue.

Design warning

These warnings and edge cases are specific to the Golden Gate design algorithm and output interpretation.

Case Trigger Current behavior in GGA module
Insert count cap More than six insert rows are attempted. The current implementation caps Golden Gate design to six inserts. The interface disables further addition once six inserts are present, and the internal collection logic also truncates to the first six non-empty inserts.
Vector has fewer than two selected Type IIS sites The chosen vector contains zero or one site for the selected enzyme. The module does not produce a full Golden Gate assembly design. The output is rendered as N/A, and the gel model shows only the uncut vector or a linearized single-cut vector.
Automatic backbone selection The vector contains multiple digest fragments. The module automatically uses the longest digestion fragment as the candidate backbone and exposes a fragment selector so the user can verify or override that choice. Changing the selected backbone also recalculates the automatic first/last insert overhangs.
Boundary overhang fields are not user-editable The first insert left overhang and the last insert right overhang are boundary conditions derived from the chosen vector backbone. Those two fields are displayed as auto-filled read-only values. If vector sequence, enzyme, or backbone choice changes, the module overwrites them with the newly derived boundary overhangs.
Vector-end overhangs not recoverable The module cannot reliably derive Type IIS overhangs at the chosen vector ends. The design falls back to a more generic seamless-rule strategy at the vector boundaries, using fragment-derived end sequences instead of vector-derived overhang constraints.
Internal junction overhangs are candidate-limited unless manually fixed If an internal junction is left blank, the algorithm only considers two native candidates: the last 4 bp of the upstream insert or the first 4 bp of the downstream insert. If the user supplies a custom 4-bp internal overlap, that manual choice replaces the native-candidate search for that junction. This keeps the default design scar-minimized and sequence-derived, while still allowing users to pin a specific internal 4-bp junction when required by a cloning standard or part library.
Homopolymer / palindromic 4-mer rejection A candidate overhang is a 4-bp homopolymer or a 4-bp palindrome. That candidate is excluded from valid overhang combinations because it is more prone to misligation or ambiguity.
No valid unique junction set found No overhang combination passes the uniqueness filter together with reverse complements, or fidelity optimization cannot identify a valid preferred set. The algorithm falls back to a default candidate choice for each junction, so a design may still be produced even when the overhang set is suboptimal.
Fidelity score unavailable The ligation-fidelity matrix cannot be loaded or computed for the selected enzyme / junction set. The design still proceeds, but the fidelity panel may be absent and the overhang set is chosen without a final matrix-based ranking.
First / last overhang fallback Vector-derived end constraints are unavailable. The first insert left overhang falls back to the 5′ end of insert #1, and the last insert right overhang falls back to the 3′ end of the terminal insert.
Internal enzyme sites inside inserts The selected Type IIS enzyme cuts within one or more insert sequences. A warning is shown because those inserts may be fragmented during the Golden Gate reaction, even if primers can still be generated.
Assembly diagram is schematic Any successful multi-fragment design. The assembly image is a conceptual schematic rather than a length-accurate plasmid drawing; users should rely on the sequence output and overhang table for exact interpretation.

Practical interpretation: in Golden Gate Assembly, a warning often reflects topology or overhang quality rather than a complete design failure. Users should interpret warnings together with the backbone selector, the overhang table, the optional fidelity section, and the final assembled sequence.
