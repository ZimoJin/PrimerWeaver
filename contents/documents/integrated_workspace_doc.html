Integrated Cloning Workspace

The Integrated Cloning Workspace provides a unified front end for four cloning workflows in PrimerWeaver: Restriction Cloning, Golden Gate Assembly, Gibson Assembly, and USER Cloning. Instead of asking users to re-enter vector and insert sequences in each individual module, the workspace keeps a shared sequence layer at the top and renders the selected method below with its native method-specific design settings and outputs.

What This Workspace Is For

This document describes only the workspace-specific integration layer: shared inputs, method switching, synchronization, persistence, and workspace-level reset or demo behavior. It does not repeat the full design logic of each cloning method.

Method-specific documentation:
  • Restriction Cloning
  • Golden Gate Assembly
  • Gibson Assembly
  • USER Cloning

Quick Start Guide

  1. Paste or upload the shared Vector sequence and Insert sequence in the top input area. These fields accept FASTA, plain text, and standard GenBank text files (.gb, .gbk, .gbff, .genbank), just like the underlying cloning modules.
  2. Select a cloning method from the Cloning method dropdown. The workspace will then render the corresponding method-specific interface below the shared inputs.
  3. Adjust the method-specific design parameters, presets, and optional advanced settings in the rendered module area, then run the design normally.
  4. Use Demo Set only after selecting a method; the workspace will load the method’s own demo content into the shared area and synchronized method pane.

Workspace Architecture

The workspace is a shared-shell adapter rather than a fifth cloning algorithm. It preserves one common sequence input region and dynamically mounts the selected cloning module below it. Only one method pane is active at a time, which prevents DOM conflicts between modules that use overlapping element IDs or method-local UI assumptions.

Layer Role in the workspace
Shared top inputs Hold the common vector and insert sequence state used across cloning methods.
Method selector Chooses which cloning module is rendered and synchronized below the shared inputs.
Method pane Renders the native interface of Restriction, Golden Gate, Gibson, or USER cloning, while hiding duplicated top-level fields already provided by the workspace shell.
Synchronization layer Keeps shared vector/insert state aligned between the workspace shell and the currently active method pane.

Shared Inputs and Synchronization

The top Vector sequence and Insert sequence areas are the canonical input fields for the workspace. When a method is selected, the workspace copies those values into the method-specific DOM and keeps the active pane synchronized as the user edits content.

This means users can compare different cloning strategies without manually re-pasting the same vector and insert sequences for every module. If the active method modifies the synchronized input fields, the shared workspace fields are also updated so that method switching remains consistent.

Practical interpretation: the workspace is designed to reduce re-entry of shared sequence input, but it does not flatten all method-specific assumptions into one universal algorithm. Final interpretation should still rely on the selected cloning method and its own downstream settings, warnings, and results.

Method Switching Behavior

Switching the cloning method does not clear the shared vector and insert draft by default. The workspace keeps those sequences available across method changes so users can compare methods on the same construct idea.

If the newly selected method supports fewer inserts than the previous interpretation of the shared insert panel, the workspace raises a confirmation prompt before truncating the effective insert set. This prevents silent loss of insert rows when moving from a more permissive assembly method to a more restrictive one.

Workspace switch behavior Current implementation
Shared draft retention Vector and insert drafts are preserved across method switches within the same session.
Insert-capacity mismatch check The workspace warns before dropping inserts when switching to a method with lower insert capacity.
Method-specific refresh hooks Active method panes can request redraws such as vector-map refresh after synchronization.

Demo Set and Reset

The workspace-level Demo Set button is method-aware. A cloning method must be selected first; otherwise the workspace shows a warning instead of guessing which module’s demo should be loaded.

The workspace-level Reset button behaves more aggressively than a local method reset. It clears the shared workspace draft and refreshes the page so that both the shell state and the active method pane return to a clean state.

Workspace-Specific Notes

  • The shared top input area is intentionally simplified to Vector plus Insert; method-specific enzyme, overlap, primer, and parameter settings stay in the method pane below.
  • Parameter presets shown in the lower method pane remain method-specific. The workspace unifies navigation and shared sequence input, but does not merge cloning parameter presets into one global preset format.
  • Vector-map behavior, warnings, and result interpretation remain the responsibility of the selected cloning module.
  • The workspace is best understood as a comparison and orchestration layer built on top of the four cloning tools, not a replacement for their method-level documentation.
